Fraud Rules

Regles de detection utilisees par le systeme

Comment fonctionne la detection ?

Le systeme analyse 741 millions d'appels et extrait des indicateurs (features) pour chaque numero de telephone. Ensuite, il applique des regles metier (definies par l'entreprise) pour identifier les comportements suspects. Un numero est considere suspect s'il correspond a au moins une regle.

Features extraites par le systeme

#FeatureDescriptionFormule SQL

Regles de detection (entreprise)

Rule Bypass General

Detecte les numeros avec 100% de variance (chaque appel vers un numero different) et zero duree entrants (ne recoit aucun appel abouti).
variance_sortants >= 100% duree_entrants = 0 appels_sortants >= 30
HAVING variance_sortants >= 100
AND duree_entrants = 0
AND appels_sortants >= 30

Rule Bypass SIM Box (IMEI)

Un meme numero utilise dans 3 appareils ou plus. Normal : 1 telephone = 1 IMEI. SIM Box : les SIM sont tournees entre plusieurs boitiers.
distinct_imei >= 3 appels_sortants >= 3
HAVING COUNT(DISTINCT imei) >= 3
AND appels_sortants >= 3

Rule Cell Location

Le numero ne bouge presque pas (tres peu de positions distinctes). Un humain normal se deplace, une SIM Box est fixe.
location_count <= 3 variance_sortants >= 85% appels_sortants >= 15
HAVING location_count <= 3
AND variance_sortants >= 85
AND appels_sortants >= 15

Rule Active Hours

Le numero est actif sur tres peu d'heures (burst d'activite). Les SIM Box operent souvent en rafale sur quelques heures.
active_hours <= 3 variance_sortants >= 100% appels_sortants >= 7
HAVING active_hours <= 3
AND variance_sortants >= 100
AND appels_sortants >= 7

Rule Bypass Offnet

Detecte les appels entrants d'un prefixe suspect avec une variance tres elevee (95%+ de numeros differents appeles).
variance_sortants >= 95% appels_entrants >= 15
HAVING variance_sortants >= 95
AND appels_entrants >= 15

Regle combinee (Liste Noire)

La regle utilisee pour creer la liste noire actuelle. Un numero est suspect si au moins une condition est remplie :
appels_sortants >= 15 + variance >= 85% OU imei >= 3 OU locations <= 3
WHERE appels_sortants >= 15
AND (variance_sortants >= 85
  OR distinct_imei >= 3
  OR location_count <= 3)

Comment le Machine Learning utilise ces regles ?

Etape 1 : Les regles ci-dessus sont utilisees pour creer des pseudo-labels (on etiquette chaque numero comme "fraude" ou "normal" selon les regles).
Etape 2 : Le modele XGBoost apprend sur 70% des numeros etiquetes.
Etape 3 : On teste le modele sur les 30% restants qu'il n'a jamais vus.
Resultat : Le modele detecte les fraudes avec F1 = 0.997 (99.7% de precision).

L'avantage du ML : il peut detecter des combinaisons de features que les regles simples ne voient pas.
